### Auto-Refresh Architecture

**File**: `src/claude-oauth-refresh.ts`

```typescript
// Automatic flow:
// 1. On each request, check if token expires in <5 minutes
// 2. If yes, use stored refresh_token to get new access_token
// 3. Cache in-memory + persist back to ~/.claude/.credentials.json
// 4. Claude Code sees the update automatically
// 5. If refresh fails, fall back to existing token
```

**Safety Features**:

- ✅ 5-minute expiry buffer prevents mid-request failures
- ✅ Graceful fallback if refresh fails
- ✅ In-memory cache for performance
- ✅ Persists refreshed token for durability
- ✅ Non-blocking: refresh happens on-demand, not scheduled

**Credential Path**: `~/.claude/.credentials.json`

- Maintained by Claude Code application
- Contains: accessToken, refreshToken, expiresAt, subscription type
- Currently shows: `max` subscription (5x rate limit)

## Resilience Strategy

### Single Provider Failure

**If Anthropic goes down**:

- OAuthRouter routes to Google Gemini
- Rate-limit fallback: `fromProviders: ["anthropic"]` → `chain: [google]`
- Current fallback model: `google/gemini-2.5-pro`

**If Google goes down**:

- OAuthRouter routes to Anthropic only
- No secondary fallback (but 5x rate limit on Claude Max)

**If both down**:

- OAuthRouter returns 502
- OpenClaw falls back to direct Anthropic key (if configured)

### Key Expiry Scenarios

| Scenario                             | Behavior                           | Recovery                                    |
| ------------------------------------ | ---------------------------------- | ------------------------------------------- |
| **Claude OAuth expires in <5m**      | Auto-refresh triggered             | Token refreshed in-memory                   |
| **Auto-refresh fails**               | Fallback to last token             | Manual refresh needed if token hard-expires |
| **Token hard-expires (no fallback)** | 401 Unauthorized → Google fallback | Health check alerts → Manual intervention   |
| **Refresh token invalid**            | Can't refresh                      | Manual intervention (re-auth Claude Code)   |
| **Google API key invalid**           | 401 Unauthorized                   | Manual key rotation required                |

## Security Considerations

### 1. Credential Storage

✅ **Safe**:

- Claude Max token in `~/.claude/` (user home, file mode 0600)
- Google API key in config.local.json
- `.gitignore` includes `config.local.json`

⚠️ **Risk**:

- If home directory is compromised, both tokens exposed
- Google key could be rotated (API key renewal process)
- Claude OAuth can't be pre-rotated (tied to Claude Code session)

### 2. Token Leakage Prevention

✅ **Implemented**:

- config.local.json NOT committed to git
- Old leaked keys were purged from git history (via git filter-repo)
- LaunchAgent & cron jobs don't log keys
- Health check report only shows key prefix (e.g., `Bearer AIzaSy...`)

### 3. Audit Trail

✅ **Available**:

- Health check reports: `~/.openclaw/oauthrouter/health-report.json`
- Server logs: `/tmp/oauthrouter.log`
- Claude Code maintains token expiry in `~/.claude/.credentials.json`

## Troubleshooting Guide

### "Unauthorized" from Anthropic

1. Check if token has expired:
   ```bash
   cat ~/.claude/.credentials.json | jq '.claudeAiOauth.expiresAt'
   ```
2. Check if refresh token is present:

   ```bash
   cat ~/.claude/.credentials.json | jq '.claudeAiOauth.refreshToken'
   ```

3. Manually trigger refresh:
   - Open Claude Code app → it will refresh the token
   - Or wait for auto-refresh in OAuthRouter (5-minute buffer)

4. Check recent OAuthRouter logs:
   ```bash
   tail -50 /tmp/oauthrouter.log | grep -i "oauth\|anthropic"
   ```

### "Unauthorized" from Google

1. Verify key is in config:

   ```bash
   grep -A 3 '"google"' ~/projects/oauthrouter/config.local.json
   ```

2. Test key directly:

   ```bash
   curl "https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent?key=AIza..."
   ```

3. If invalid: Rotate the key
   - Generate new key at https://aistudio.google.com/apikey
   - Update config.local.json
   - Restart OAuthRouter

### Health Check Failing

1. Run manually:

   ```bash
   node ~/projects/oauthrouter/scripts/health-check.mjs
   ```

2. Check Claude credentials file:

   ```bash
   ls -lah ~/.claude/.credentials.json
   cat ~/.claude/.credentials.json | python3 -m json.tool
   ```

3. Check if OAuthRouter is running:

   ```bash
   lsof -i :8099
   ps aux | grep oauthrouter
   ```

4. Restart OAuthRouter:
   ```bash
   launchctl unload ~/Library/LaunchAgents/com.oauthrouter.plist
   launchctl load ~/Library/LaunchAgents/com.oauthrouter.plist
   ```
